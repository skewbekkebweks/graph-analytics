# Тестовое задание «Аналитика над графом»

## Задание

Дан невзвешенный, ориентированный граф. Требуется реализовать **один из** существующих аналитических алгоритмов на графах, при условии, что весь граф не помещается в доступную оперативную память.

## Ключевая сложность

1. Граф **не помещается в оперативную память** _единственного_ вычислительного узла. Например, доступная оперативная память 128 Мб, тогда как для всего графа необходимо 1 Гб.
2. Граф **может** содержать гипер-узлы, т.е. вершины со степенью на несколько порядков больше средней. Например, средняя степень вершины равна 50, гипер-узел будет иметь степень 50'000.

Необходимо выбрать **эффективный способ представления и обработки графа**.

## Этапы выполнения задания

1. **Выберите одну из аналитических метрик для графа, какую посчитаете нужным.** Мы рекомендуем использовать: *PageRank, LeaderRank, Katz, Jaccard, Dice*, но допустимо взять любую другую аналитическую метрику, т.е. метрика которая применяется ко всему графу.
2. **Изучите выбранную метрику**
   a. Разберитесь как работает метрика и где может быть полезна
   b. Кратко аргументируйте выбор метрики
3. **Предложите несколько (не более трёх) вариантов** эффективного представления и обработки графа (с обоснованием выбора) для вычисления выбранной метрики
4. **Реализуйте один из них** в коде
5. **Опишите в коде или документации**, почему выбран конкретный метод и какие есть ограничения у данного подхода

## Критерии оценки

- Эффективность работы с большими графами (граф не влезает в память!)
- Корректность работы алгоритма (возможность завершения программы за разумное время)
- Параллельная обработка данных
- Чистота кода

## Правила

- Убедитесь, что результатам можно доверять. Исключите вариант случайности, etc.
- Присылайте решение в виде репозитория на github с отчетом по решению и чёткими инструкциями, как запустить ваш код. **Убедитесь, что мы сможем запустить ваше решение по этим инструкциям.**
- Вы можете использовать любые библиотеки и фреймворки, которые вам могут быть необходимы. **Однако выбранный алгоритм и работа с памятью должны быть реализованы в вашем коде.**
- Вы можете использовать Google Colab или Kaggle Code, чтобы получить доступ к бесплатным вычислительным ресурсам.
- Вы можете использовать готовые графовые датасеты, например, SNAP или LDBC SNB.
- Предпочтительно писать программу на языке C++ или Java, но в целом можно выбрать любой. В случае выбора языка отличного от рекомендуемого, предоставьте дополнительную инструкцию о настройке среды или контейнер для сборки.
- Сфокусируйтесь на том, чтобы код был чист и понятен. Если вы считаете, что какая-то его часть может быть непонятна, то добавьте комментарии. Мы очень сильно ценим хорошо написанный код, поэтому если решение задачи будет оформлено грязно, то мы можем отклонить заявку.
- **Используйте многопоточность.** Вычислительный узел только один, но необходимо максимально использовать **все его доступные CPU**.
- **Платформа:** x64
- **Рекомендуемая ОС для стенда запуска:** Ubuntu 22.04
- **Дисковая подсистема:** SSD

## Советы по выполнению задания

- Если столкнулись с трудностями, можете спросить у ChatGPT, главное понять и суметь рассказать, что было сделано.
- Оцените вычислительную сложность и точность вашего алгоритма в худшем и среднем случаях. Если возможно, докажите важные свойства вашего алгоритма.
- Для представления входных и выходных данных на файловой системе можете использовать CSV или колоночные форматы, например, Parquet.

## Пример входных данных

- Формат: **CSV** (первые 2 колонки — id пары вершин `from` и `to`), описывающих ребро из from в to.

  Формат вершин — int32.

- Рёбра не имеют веса.
- Содержимое входного файла (`edges.csv`):

```
from,to
1,2
2,3
3,1
4,1
```

## Пример выходных данных

- Формат: **CSV**
- Формат вершин — int32
- Пример выходного файла (`leader_rank.csv`):

```
vertex,rank
1,0.35
2,0.25
3,0.25
4,0.15
```
